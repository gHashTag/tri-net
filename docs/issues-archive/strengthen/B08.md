## Goal
Add a bit-exact GF16 (φ-derived 16-bit float) host reference model of the OFDM FFT/equalizer datapath — a new `src/gf16.rs` module — so the Zynq PL demod can be prototyped and conformance-checked in Rust before any RTL/hardware exists.

## Why / science
Fitting the OFDM demodulator (FFT + per-subcarrier equalizer taps) into the Mini's **Zynq-7020 PL (~85K logic cells / ~220 DSP48)** is the top greenfield tech risk in the project. GF16 is **silicon-proven at 323 MHz on Artix-7 (XC7A35T), passing a 35-of-35 FPGA-codec testbench** — the same FPGA family as our proven AX7203 board. Because GF16 is **16-bit vs fp32's 32-bit**, the realistic win is *area / multiplier-width* (more parallel taps per DSP48), **not** accuracy — the GoldenFloat paper explicitly makes **no per-rung accuracy or superiority claim**, so this issue must validate against externally-generated conformance vectors, not assert correctness.

This module is a **host model only**: it maps to the Verilog that `t27` will later emit, giving M2's demod work a testable, cross-checked reference. It does not alter the M1 crypto/routing/wire paths.

Citation: **arXiv:2606.05017** (GoldenFloat; GF16 codec at 323 MHz on XC7A35T, 35/35 testbench). Reference repo `gHashTag/zig-golden-float`. **FFI note:** the committed `rust/goldenfloat-sys` `extern "C"` surface exposes `gf16_from_f32`/`gf16_to_f32`/`gf16_add`/`gf16_mul`/**`gf16_fma`** (and `gf16_phi_quantize`/`gf16_phi_dequantize`) — it does **not** currently export `gf16_phi_dot`/`gf16_phi_fma` (those appear only in the Zig C-ABI docs). So the FFI path must build `phi_dot`/`phi_fma` from the real `gf16_fma`/`gf16_mul`/`gf16_add` symbols; `phi_dot`/`phi_fma` are our own composed primitives. Cross-check vectors generated offline with `ml_dtypes`.

## Tasks
- [ ] Add a `gf16` module to `src/lib.rs` (alongside `crypto`/`routing`/`wire`/`discovery`/`daemon`) and re-export the public type(s). Keep `#![forbid(unsafe_code)]` intact — if `goldenfloat-sys` FFI is wired in, isolate the `unsafe` FFI in a leaf crate/adapter and expose only a safe wrapper (see feature-flag task), so `trios-mesh` itself stays unsafe-free.
- [ ] Create `src/gf16.rs` implementing a `Gf16` scalar type (decode↔f32, add, mul, `fma`) as the bit-exact reference, mirroring the GF16 format from arXiv:2606.05017 (`[s:1][e:6][m:9]`, bias 31, round-to-nearest-even, no subnormals). Pure-Rust reference path is the default (no FFI dependency for `cargo test`).
- [ ] Implement `phi_dot(a: &[Gf16], b: &[Gf16]) -> Gf16` and `phi_fma` **as our own primitives** on top of `Gf16` add/mul/fma, used as the tap-accumulation primitive. Document the accumulation order so it maps deterministically to future RTL.
- [ ] Implement a radix-2 (or radix-4) **FFT** and a per-subcarrier **1-tap equalizer** over `Gf16`, sized for a small OFDM symbol (e.g. N=64), expressed only in terms of `phi_dot`/`phi_fma` + `Gf16` ops so it maps to future `t27` Verilog.
- [ ] Vendor a conformance-vector file under `tests/vectors/gf16_ofdm.json` (or `.csv`) generated **offline** by an `ml_dtypes`-cross-checked script; commit the generator under `scripts/` and document the exact command in a header comment. Do not hand-author expected values.
- [ ] Add `tests/gf16_conformance.rs` that loads the vectors and asserts the FFT/equalizer output matches bit-exactly (or within the format's defined ULP tolerance, documented in-file).
- [ ] Add an **optional** `goldenfloat-ffi` Cargo feature (default off) that swaps the pure-Rust `Gf16` scalar ops for the `goldenfloat-sys` FFI (`gf16_add`/`gf16_mul`/`gf16_fma`/`gf16_from_f32`/`gf16_to_f32`); `phi_dot`/`phi_fma` stay composed in Rust over whichever scalar backend is active. Add a test `ffi_matches_pure_rust` asserting both paths agree on the shared vector set. CI runs the default (pure-Rust) path.
- [ ] Update `README.md` module table with a `gf16` row marked milestone **M2 (host model, `-sim`)**, and a one-line note that GF16's win is width/area, not accuracy.
- [ ] Do **not** touch `src/crypto.rs`, `src/routing.rs`, `src/discovery.rs`, `src/wire.rs`, or `src/daemon.rs` logic beyond the `lib.rs` module declaration/re-export.
