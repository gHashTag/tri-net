Один примитив, понятными образами

Троичный вес = светофор для умножения

Умножить на {−1,0,+1} — не арифметика, а выбор: пропустить +x, инвертировать −x, или стоп 0. Ни одного DSP-умножителя.

Это и есть балыбердинское «нет-данных» = красный свет: узел молчит, пока операнд не пришёл.

Систолическая волна = волна на стадионе

Данные катятся фронтом по решётке процессоров; каждое «кресло» (PE) срабатывает один раз, когда волна проходит, читая из своей локальной памяти.

Дословно его «волна детонации по сети ССИ» — только у нас на одном чипе, а у него между модулями в здании.

Стена памяти = одна касса на супермаркет

Классический процессор гоняет всё через одну общую ОП — очередь у единственной кассы. Наш замер: 23 МБ весов на токен упираются в DDR.

Dataflow даёт каждому ряду свою кассу (локальная память PE) — очередь исчезает. Это его главный тезис.

CDMA-меш = коктейльная вечеринка

Много узлов говорят в одной полосе; слушатель выделяет своего по «коду» (как язык собеседника в шуме). Доказано: два узла, BER=0 каждый.

Детерминированное разделение без взаимного влияния клиентов — прямое требование ССИ.

Отчёт за неделю · что построено (с образами)

  • Радио заговорило. Чистый байт, затем читаемый текст «TRINET ternary OTA link» по реальному эфиру 2.4 ГГц, BER=0 / 576 бит. Приёмник: M²-оценка несущей + согласованный фильтр по 819-чиповой преамбуле + Costas. Образ: сначала научились держать ноту (несущую), потом — слоги (символы), потом — слово без единой ошибки.
  • Два узла в одной полосе (CDMA). .11 и .13 передают одновременно; .12 разделяет по коду, BER=0 каждый. Охарактеризован near-far эффект. Образ: вечеринка — оба говорят разом, но каждый на своём «языке-коде», и слушатель понимает обоих.
  • Радио И ИИ на одном чипе. AD9361 ловит эфир на PL, троичный классификатор считает на ARM того же кристалла — не симуляция, живое радио не тронуто. Образ: одно ухо слышит, тот же мозг тут же понимает — на одном «нейроне» за $100.
  • Обученная модель сети. 5 классов эфира (шум / глушилка / узел-A / узел-B / чужой широкополосный), собранных С РЕАЛЬНОГО эфира; float 99.8% = троичная 99.8%, вся модель 52 байта. Образ: узел научился по «голосу» эфира понимать, кто говорит и свободен ли канал.
  • ИИ стал медиа-доступом (CSMA). Узел слушает моделью → «чисто → передаю», «занято (глушилка/узел-B) → жду». Вживую 3/3. Образ: вежливый собеседник — не перебивает, ждёт паузы. Это МЭК-61499-событие «канал свободен».
  • Троичный трансформер и код-LM на ARM. Self-attention + FFN, все веса {−1,0,+1}; генерирует валидный код (fn main(){let x=phi*phi+1;}) на самом узле. Образ: архитектура 91M IGLA-Coder в спичечном коробке — та же голова, что у больших моделей.
  • Открытый флоу до кремния. PS7-периферал целиком через yosys→nextpnr→fasm2frames→xc7frames2bit → загружаемый битстрим 4.05 МБ @ 308 МГц, без Vivado. Образ: суверенный путь «спека → кремний» без единой западной проприетарной кнопки.
  • Честная цена троичности измерена. На 2 слоях троичность теряет точность (float 87% vs ternary 55%) — тот разрыв, что BitNet закрывает масштабом. Образ: показали не только где летает, но и где пока падает — карта без прикрас.

Слабые места (IGLA-аудит, честно)

  • Троичность на 2+ слоях теряет точность (87→55%). Нужны BitNet-приёмы (масштаб данных, learned scale, длинный QAT) — иначе «настоящая» кодовая модель на узле не дотянет.
  • 91M IGLA-Coder — пока архитектура + бюджет, не обученный прогон. Мостик к нему (обученный троичный трансформер в миниатюре) есть, полного обучения нет.
  • Радио — стендовая пара, антенна-в-антенну. Дальность/многолучёвка/N-узловой эфирный меш не измерены. Клок-recovery для АЦП→ЦАП (сценарий Балыбердина) в железе ещё не собран.
  • PS7-битстрим собран, но не прошит на кремний с реальными пинами + PS-софтом — деструктивный шаг под физическим присутствием.
  • Мы держим ALU/радио-слой, не сетевой fabric ССИ и не polyhedral-математику ЯПФ (этап 5 Балыбердина) — это его 5 лет работы, наша сторона комплементарна, не дублирует.
  • gen/ репозитория tri-net перманентно «грязный» (t27c дрейфанул), сборка только рядом с t27 — техдолг тулчейна.

  2. ASU_TP_SOM на нашем железе. ✓ узел-битстрим собран. asutp_som.v: PS7 (вычислитель+управление) через EMIO-GPIO (интерфейс виртуального канала ССИ) рулит операндами/весами/событием fb61499; clkrec даёт детерминированный такт. Открытый флоу → загружаемый asutp_som.bit 4.05 МБ @ 106 МГц. RISC-V soft-core + 80 I/O — дальше.
  3. Клок-recovery по сценарию АЦП→ЦАП. ✓ RTL, 122 МГц. NCO+FIFO+PI (clkrec.v): в iverilog идеальный захват (ошибка 0.000000%, буфер на 50%), пост-P&R на xc7z020 = 122 МГц. Детерминированная петля восстановления такта в железе — приёмка AC-2 (МСЭ-Т G.8261).
  4. Загружаемая топология ЯПФ. ✓ RTL 2026-07-18. yapf_fabric.v: N троичных функциональных блоков, чей граф связей ПРОГРАММИРУЕТСЯ конфиг-регистрами в рантайме (что PS пишет по EMIO), а не запечён в битстрим — прямой ответ на «загрузить карту связей / править структуру соединений на ходу». Один битстрим, три загруженных топологии дают 18 · 0 · 2 (iverilog). Закрывает пробел «статичная топология»: узел программируем на уровне графа, не только параметрами.
  5. ИИ-слой поверх ССИ. Наш обученный троичный сенсинг/классификатор как «интеллект узла» для маршрутизации, детекции аномалий, приоритезации (coordinated service SG-13).
  6. $TRI DePIN — экономика узла. ✓ на ARM узла 2026-07-18. specs/tri_depin.t27: Proof-of-Relay — подделко-стойкий аккумулятор пересланного трафика (аналог Helium Proof-of-Coverage), из которого слой расчётов чеканит $TRI. Печать связывает и историю, и количество байт (закрыта дыра инфляции награды). 14 инвариантов через golden pipeline. Прогнан на реальном железе сквозняком (A+B+C): узел .13 отдаёт данные, узел .12 тянет их по сети, считает чек и подписывает Ed25519; host (x86) и второй узел .13 проверяют публичным ключом, без общего секрета; расчётный слой tri_settle.t27 делит пул $TRI пропорционально байтам, не превышая пул. Инфляция счётчика и чужой ключ — отклонены. Радиоканал измерен: .13 TX-тон → .12 RX даёт SNR 47.4 дБ (BPSK BER≈0 — байты дойдут точно); а для худших линков — интегрити-гейт (метит только при совпадении дайджеста), XOR-FEC + перемежитель (tri_fec.t27 + tri_ilv.t27: битая датаграмма восстанавливается, а перемежитель раскидывает соседей по кодсловам, так что фейдинг-пачка длиной ≤8 переживается — на ARM пачка 8: обычный FEC теряет 2 кодслова, перемежённый восстанавливает всё 64/64; адаптивная глубина choose_depth — узел подбирает глубину под свою пачку: пачка 12 — фикс. D=8 теряет, D=12 спасает всё) и полный живой DePIN-раунд по РЕАЛЬНО измеренному покрытию (tri_settle.t27: 3 узла на реальных замерах .13→.11/.12 при разной мощности = 34/18/6 дБ, агрегация эпох + взвешенная выплата на ARM — узел0 60 КБ@34дБ → 713 $TRI, узел1 45 КБ@18дБ → 258, узел2 24 КБ@6дБ → 27, paid ≤ пул; живой Helium-класс Proof-of-Coverage). Осталось: полный OTA-перенос байт через модем (демод в петле BER=0, поднесущая обходит DC-блок AD9361; блокирует выкладка буфера в TX) и живой демон меша. Стимул строить физическую сеть — то, чего нет в стандарте, но что запускает развёртывание «снизу вверх».
  7. Суверенная воспроизводимость. Наш открытый битстрим + φ-anchor/SHA-цепочка как R-TT reproducibility для его прототипа: инвестор/госзаказчик видит криптопривязку spec→bitstream.

Где это в 6G — честный маппинг на IMT-2030 (МСЭ-R, июнь 2023)

МСЭ-R задаёт радио-интерфейс IMT-2030 (6 сценариев), МСЭ-T SG-13 — сетевую архитектуру (Сеть-2030). TRI-NET ложится на оба слоя. Честно: попадаем в 3 из 6 сценариев напрямую, ещё в 1 частично; токен $TRI и суверенный открытый флоу — комплементарный слой поверх стандарта, не пункт самого стандарта.
